Learning Objectives

The primary objective of this course is to introduce the basic concepts and practices of computer science. The core ideas of computer programming will be explored using the Python programming language. We will cover fundamental approaches to problem solving and the creation of software solutions as well as introduce basic computational concepts

Computer science students will gain a foundation for further study if computer science, including a more in-depth treatment of object-oriented programming in CIS 211. Students in other majors will find rich connections from computational concepts to many fields. Learning to program is also an excellent way to build and exercise problem solving skills and gain access to a powerful problem solving tool.

Understand ...

Sequential execution model

Have a reasonably accurate, useful mental model of program state, including local variables in the stack and references to values in the heap. Understand the difference between a name, a variable, a memory cell, and the value stored in a memory cell. Understand how two variables could refer to the same memory cell (aliasing), and how two variables could have the same name (in different scopes).

Understand program execution as a series of modifications to program state. Be able to sketch the effect of simple program statements, and to accurately predict the effect of a sequence of program statements.

Demonstrated by ..

  • Construction of projects
  • Exam questions that require predicting the effect of code.

Control Flow

Operationally understand conditional control flow, including nested conditionals, short-circuit evaluation, and sequential composition of conditional statements. Be able to accurately predict the effect of program fragments including conditionals.

Operationally understand loops, including loops over aggregate data structures (“for each X in S”), while loops, and for loops.

Have a basic problem-solving vocabulary of loop patterns involving aggregate data types, including summarizing (summing, counting, etc.), filtering, and sequential search. Be able to choose an appropriate loop pattern to solve basic programming problems.

Be able to design and implement control flow involving nested loops and conditional execution.

Be able to design inductive (recursive) loops, explicitly identifying base cases and progress cases, and reason about termination.

Demonstrated by ..

  • Construction of projects that require design of loops, including binary search and non-trivial loops
  • Exam questions that require predicting the effect of idiomatic loops, and composition of idiomatic and more challenging loops.

Procedural Abstraction

Have a reasonably accurate, useful mental model of the effects of program call and return on program state. Understand how a procedure call creates a new activation record containing local variables, how parameters are passed to procedures, and how procedure return discards the activation record.

Understand parameter passing both from a programming perspective (e.g., copying actuals into formals by position rather than name) and at a more detailed operational level (e.g., how passing a reference type by value has the effect of aliasing). Be able to accurately predict the effects of program sequences involving procedure call and return. Understand that modifications to the heap persist beyond procedure return, while modifications to procedure-local variables don't.

Understand the difference between local and global variables, and be able to predict the effect of modifications to either.

Understand procedural abstraction as a design tactic to break larger problems into brain-size chunks. Understand that some decompositions into procedures are better than others, and begin to develop some skill in making good decomposition decisions.

Understand that procedures can call other procedures, and be able to design programs with multiple levels of procedure calls.

Demonstrated by ..

  • Completing projects that require breaking a complex problem down into multiple functions.
  • Exam questions that require predicting the effect of multiple levels of function calls with arguments that may or may not share references.

Testing and Debugging

Understand that testing is part of programming, and can be systematic rather than haphazard.

Understand that debugging is part of programming, and can be systematic rather than haphazard.

Demonstrated by ..

  • Completing projects of sufficient complexity that debugging will almost certainly be required during development.

Programming Style

Understand that design is part of programming, and can be systematic rather than haphazard

Ability to design and write code that reflects awareness of standard design techniques, good programming style, Python style standards, and supports testing.

Demonstrated by ..

  • Completing projects according to class design and programming style specifications.

Representation and Interpretation

Understand that the same memory representation (bit pattern) may be interpreted in different ways. Understand the notion type as indicating the intended interpretation and potential operations on a value in memory. Understand why the string “1” is different from the integer 1 and from the floating point value 1.0.

Understand the difference between numbers and computer representations of numbers. Understand that computer operations like addition and division do not obey all the familiar mathematical properties of real numbers, because they operate on computer representations rather than actual numbers.

Demonstrated by ..

  • Exam questions that require predicting the effect of code manipulating numbers, strings, and other objects.

Computational Complexity

Understand, at a basic level, how to reason about the relation between the number of computational steps and problem size (time and space).

Demonstrated by ..

  • Exam questions that require composing algorithms that executes within a given complexity bound (typically linear time).
